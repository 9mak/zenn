## 🚀 はじめに

[おぐま](https://github.com/9mak)です。

最近、AIツールに「定期実行」機能がどんどん追加されてきていて、ちょっとしたタスクの自動化が身近になってきました。Difyのスケジュールトリガー、Claudeのスケジュールタスク、GeminiのScheduled Actions。どれも「AIを定期的に走らせる」という機能なんだけど、仕組みや使い勝手がけっこう違うので、実際に比べてみました。

## 📊 比較表

| 項目 | Dify | Claude | Gemini |
| --- | --- | --- | --- |
| 機能名 | スケジュールトリガー | Scheduled Tasks（Cowork / /loop） | Scheduled Actions |
| リリース時期 | 2026年初頭（v1.10.0） | 2026年2月 | 2025年6月 |
| 最小実行間隔 | cron式で分単位まで | Cowork: 1時間〜 / /loop: 1分〜 | 1日1回 |
| cron式サポート | あり（5フィールド） | /loopのみあり | なし |
| 外部ツール連携 | API・プラグイン経由 | MCP・プラグイン経由 | Google Workspace中心 |
| 料金 | 無料（セルフホスト）/ $59〜（Cloud） | $20〜（Pro以上） | $19.99〜（AI Pro以上） |
| 実行環境 | サーバーサイド（常時稼働） | PCが起動中のみ | クラウド（オフラインでも実行） |
| 向いている用途 | ワークフロー自動化・業務処理 | 個人のタスク自動化・開発作業 | Google Workspace連携・日次ルーティン |

## 🔍 詳細比較

### Dify — ワークフローに組み込むスケジューラー

Difyはv1.10.0で「トリガー」システムを導入して、ワークフローのStartノードにスケジュールトリガーを設定できるようになりました。

**セットアップ方法**

ワークフローエディタでStartノードにスケジュールトリガーを追加するだけ。ビジュアルピッカーで「毎時・毎日・毎週・毎月」を選ぶか、cron式を直接書くこともできます。設定すると次の5回分の実行予定時刻がプレビュー表示されるのが地味に便利。

```
# cron式の例：平日の9時〜17時に15分ごとに実行
*/15 9-17 * * 1-5
```

**良いところ**

- サーバーサイドで動くので、PCの電源を切っていても実行される
- cron式で細かいスケジュール制御ができる
- ワークフロー全体と組み合わせて複雑な処理が可能
- セルフホストなら無料で制限なし

**気になるところ**

- 1ワークフローにスケジュールトリガーは1つだけ
- Dify Cloud無料プランだとトリガーイベントが3,000回/月の制限あり
- セットアップにはDify自体の構築が必要で、手軽さでは劣る

:::message
v1.10.0より前のDifyにはネイティブなスケジュール機能がなかったので、GitHub ActionsやMake.comなど外部ツールでAPI呼び出しを定期実行する方法が主流でした。古いバージョンを使っている場合はその方法も検討してみてください。
:::

### Claude — デスクトップとCLIの2つの入口

Claudeのスケジュール機能は2つのアプローチがあります。Cowork（デスクトップアプリ）の`/schedule`と、Claude Code（CLI）の`/loop`です。

**Cowork（デスクトップアプリ）でのセットアップ**

Coworkで新しいタスクを作って`/schedule`と入力するだけ。Claudeが対話形式で名前・内容・頻度（毎時・毎日・平日のみ・毎週・オンデマンド）を聞いてくれます。サイドバーの「Scheduled」セクションから管理もできます。

**Claude Code（CLI）でのセットアップ**

```bash
# 1時間ごとにissueをチェック
/loop 1h "新しいissueがないか確認して"

# cron式も使える（5フィールド）
# 分 時 日 月 曜日
```

**良いところ**

- 自然言語で設定できるのがとにかく楽
- Slack、ファイル操作、Web検索など接続したツールをフル活用できる
- /loopならcron式で細かいスケジュールも可能
- 朝のブリーフィング、週次レポートなど実用的なユースケースが豊富

**気になるところ**

- PCが起動していてアプリが開いていないと実行されない（これが一番大きい制約）
- /loopは3日で自動削除される
- /loopはClaude Codeのセッション中のみ有効（再起動でリセット）
- 実行するたびにプランの利用枠を消費する

:::message alert
Coworkのスケジュールタスクは、PCがスリープ中やアプリが閉じている間はスキップされます。復帰後に1回だけ実行されるので、「毎時」で設定しても不在中の分はまとめて1回になる点に注意。
:::

### Gemini — Google Workspaceとの連携が強み

GeminiのScheduled Actionsは2025年6月にリリースされた機能で、自然言語でスケジュールを設定できるのが特徴です。

**セットアップ方法**

チャットに「毎朝8時にカレンダーの予定とメールのまとめを送って」と入力するだけ。Geminiが自動的にスケジュール設定と判断して、確認サマリーを表示してくれます。左上の「Scheduled Actions」タブから手動で設定することもできます。

**良いところ**

- セットアップが圧倒的に簡単（チャットで指示するだけ）
- クラウド実行なので、PCがオフラインでも動く
- Gmail・Calendar・Drive・Docsとのネイティブ連携が強力
- 結果がメールやプッシュ通知で届く

**気になるところ**

- 最小実行間隔が1日1回（1時間ごとなどは不可）
- 同時にアクティブにできるスケジュールは最大10個
- cron式のような細かいスケジュール設定はできない
- Google Workspace以外のツールとの連携は限定的
- 「隔週の火曜日」のような複雑なパターンは非対応

## 🎯 どっちを選ぶ？

### サーバーサイドでガッツリ自動化したい → **Dify**

ワークフローの中にスケジュールを組み込めるので、「データ取得→加工→通知」みたいな複雑な処理を定期的に回したい場合に向いています。セルフホストなら無料だし、cron式で分単位の制御もできる。ただし環境構築のハードルはあります。

### 個人の作業を自動化したい・開発者向け → **Claude**

朝のブリーフィング、週次レポート、開発中のissue監視など、個人の生産性を上げるタスクに向いています。自然言語で設定できるし、MCP経由でいろんなツールと繋がれる。ただしPCが起動している必要がある点は要注意。

### Google Workspace中心の生活をしている → **Gemini**

Gmail・Calendar・Driveを日常的に使っているなら、Geminiが一番シームレス。クラウド実行なのでPCの電源も関係ない。ただし実行頻度が1日1回までなのと、Google以外のツールとの連携は弱めです。

:::details 3つを併用するのもあり
用途によって使い分けるのが現実的かもしれません。たとえば「朝のメール・カレンダーまとめはGemini」「開発系の監視タスクはClaude」「業務ワークフローの定期実行はDify」みたいな組み合わせです。
:::

## まとめ

3つのサービスを比べてみると、それぞれ狙っているレイヤーが違うことがわかります。

- **Dify**: ワークフロー自動化プラットフォームとしてのスケジューリング
- **Claude**: パーソナルAIアシスタントとしての定期タスク
- **Gemini**: Google Workspaceのルーティン自動化

どれが「一番いい」というよりは、自分の使い方に合ったものを選ぶのが大事ですね。個人的には、Claudeの自然言語で設定できる手軽さと、Difyのサーバーサイド実行の安定感がそれぞれ魅力的に感じました。Geminiは「Googleに閉じてもいい」ならめちゃくちゃ楽です。

定期実行機能はどのサービスもまだ発展途上な感じなので、今後のアップデートにも注目していきたいところです。
