## 🚀 はじめに

[おぐま](https://github.com/9mak)です。

ServiceNowのCAD（Certified Application Developer）試験を受けようと思い立ち、試験範囲と受験前提を一通り調べました。
とりあえず申し込めばいいかと思っていたら、思ったよりも準備が必要だったのでまとめておきます。

## 📊 調査・分析結果

### 試験の基本情報

| 項目 | 内容 |
| --- | --- |
| 試験名 | Certified Application Developer（CAD） |
| 問題数 | 60問（多肢選択・複数選択） |
| 試験時間 | 90分 |
| 合格ライン | 非公開（公式に「必ずしも70%とは限らない」と明記） |
| 受験費用 | $300 USD |
| 試験言語 | 日本語対応あり |

受験はPearsonのテストセンター（全国各地）またはオンライン（OnVUE）から選べて、登録後90日以内に予約・受験を完了する必要があります。

### 試験ドメインと出題割合

| ドメイン | 日本語 | 出題割合 |
| --- | --- | --- |
| Designing and Creating an Application | アプリケーションの設計と作成 | 20% |
| Application User Interface | アプリケーションUI | 20% |
| Security and Restricting Access | セキュリティとアクセス制限 | 20% |
| Application Automation | アプリケーション自動化 | 20% |
| Working with External Data | 外部データの操作 | 10% |
| Managing Applications | アプリケーションの管理 | 10% |

主要4分野（設計・UI・セキュリティ・自動化）が各20%で均等に並び、残り2分野が各10%という構成です。
特定の分野を集中的に勉強して乗り切れるタイプではなく、プラットフォーム開発の全体像を幅広く問われます。

:::details 各ドメインの詳細（公式ブループリントより）

**1. アプリケーションの設計と作成（20%）**
- アプリケーションがServiceNowに適しているかどうかの判断
- データモデルの設計と実装
- モジュールの作成
- アプリケーションスコープの使用

**2. アプリケーションUI（20%）**
- フォームの作成・デザイン・カスタマイズ
- フォームやテーブルへのフィールド追加・削除
- クライアントサイドスクリプトの作成・テスト・デバッグ（デスクトップ向け）
- サーバーサイドスクリプトの作成・テスト・デバッグ
- レコードプロデューサーをアプリケーションのUIとして使用

**3. セキュリティとアクセス制限（20%）**
- アプリケーションおよびアプリケーションモジュールへのアクセス制限
- アクセス制御の手動・自動での作成・テスト・デバッグ
- GlideSystemメソッドを使ったセキュリティのスクリプト化
- アプリケーションスコープを使ったアプリケーション成果物の保護

**4. アプリケーション自動化（20%）**
- ワークフローおよびFlow Designerの作成・テスト・デバッグ
- アプリケーションプロパティの作成と使用
- イベント・スケジュールジョブ・スクリプトインクルードの作成
- メールの送受信

**5. 外部データの操作（10%）**
- CSVまたはExcel形式でのデータインポート
- RESTを使った外部データソースとの統合・テスト・デバッグ

**6. アプリケーションの管理（10%）**
- アプリケーションのダウンロードとインストール
- 委任開発を使ったソースコード・コードレビューの管理
- ServiceNow Git統合を使ったソースコード管理

:::

### 受験前提・必要な準備

公式が推奨するコースが3本あります。

- ServiceNowへようこそ
- ServiceNowにおけるスクリプト作成の基礎
- アプリケーション開発の基礎

:::message alert
講師主導トレーニング（ILT）を修了すると試験を1回無料で受験できます。
オンデマンドコースでもバウチャー取得は可能なので、スケジュールに合わせて選んでください。
:::

必須の前提試験はなく、CSAを先に取得していなくても受験可能です。
ただし、以下の知識・経験は実質的に前提になっています。

- App Engine Studioの操作
- テーブル・フィールドの設計
- ACLの設定
- Flow Designerでのフロー作成
- アップデートセット・ソース管理の概念

実務経験は6ヶ月程度が目安とされていて、いきなりゼロから挑むのはかなりきつそうです。

## 🔍 考察

設計・UI・セキュリティ・自動化の4分野が各20%で均等に並ぶ構成なので、特定の得意分野で乗り切れるタイプの試験ではないとわかります。
どこか一分野が弱いとそのまま合否に響いてくるので、満遍なく勉強する必要があります。

CSAが必須ではないとはいえ、テーブル設計やACLなどCSAで問われる範囲と重なる知識が多いので、ServiceNow未経験でCADから入るのはかなりしんどいですね。

:::message
CSAを先に取ってからCADに進む流れが無難で、試験範囲の重複が多い分、CADの勉強がスムーズになります。
:::

## 💡 学んだこと・発見

- バウチャーが必要という点は見落としがちなので、受験を決めたらまず研修の予約から入る必要がある
- 試験範囲を見る限り、Flow DesignerとACLの理解が合否に最も影響しそう
- $300という費用はそれなりに高いので、一発合格を狙う価値がある

## 🎯 まとめ

- 60問・90分・合格ライン非公開・$300
- 受験にはバウチャー（研修受講が必要）
- 試験範囲は設計・UI・Security・Automationが各20%の均等配分
- CSA先取りが実質的な近道

これから対策を始めるので、合格体験記もいずれ書けたらと思います。
